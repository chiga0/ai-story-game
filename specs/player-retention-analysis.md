# 玩家流失节点分析报告

## 一、概述

### 1.1 分析目的
识别 AI Story Game 中玩家可能流失的关键节点，分析流失原因，提出优化建议，提升用户留存率。

### 1.2 分析方法
- 用户流程分析
- 关键节点假设
- 数据验证方案

### 1.3 核心指标定义

| 指标 | 定义 | 目标值 |
|-----|-----|-------|
| 次日留存 | 第2天再次打开应用的用户占比 | > 40% |
| 7日留存 | 第7天仍然活跃的用户占比 | > 20% |
| 完成率 | 开始游戏后达成结局的用户占比 | > 60% |
| 游戏时长 | 单次游戏会话时长 | > 10分钟 |

---

## 二、用户流程概览

```
首页 → 剧本列表 → 剧本详情 → 开始游戏 → 游戏进行中 → 达成结局 → 分享/重玩
  │        │          │          │            │            │
  │        │          │          │            │            └─ 流失节点 6
  │        │          │          │            └─ 流失节点 5
  │        │          │          └─ 流失节点 4
  │        │          └─ 流失节点 3
  │        └─ 流失节点 2
  └─ 流失节点 1
```

## 三、关键流失节点分析

### 节点 1：首页流失

#### 场景描述
用户首次打开应用，看到首页后立即离开。

#### 流失率预估
**30-40%**（行业基准）

#### 流失原因分析

| 原因 | 权重 | 分析 |
|-----|-----|-----|
| 价值传达不清 | 高 | 用户不理解产品是什么、能做什么 |
| 视觉吸引力不足 | 中 | 首页设计缺乏亮点 |
| 加载过慢 | 中 | 首屏渲染时间过长 |
| 无明确引导 | 高 | 用户不知道下一步该做什么 |

#### 优化建议

1. **价值传达优化**
   - 首屏明确展示核心价值：「AI 驱动的沉浸式剧本杀」
   - 添加 15 秒演示视频或 GIF 动图
   - 突出「立即体验」按钮

2. **视觉优化**
   - 动态背景效果（如打字机效果文字）
   - 剧本封面使用更吸引人的插画风格
   - 添加角色动画元素

3. **性能优化**
   - 首屏 SSR 渲染
   - 图片懒加载 + 占位图
   - 骨架屏过渡

4. **引导优化**
   - 新手引导流程（当前已有 OnboardingGuide）
   - 首次访问自动弹出热门剧本推荐

#### 数据埋点需求

```typescript
{
  event: 'home_page_view',
  session_id: string,
  is_new_user: boolean,
  load_time_ms: number,
  scroll_depth: number,
  time_on_page_ms: number,
  exit_action: 'close' | 'navigate' | 'timeout'
}
```

### 节点 2：剧本列表流失

#### 场景描述
用户进入剧本列表页，浏览后未选择任何剧本就离开。

#### 流失率预估
**20-30%**

#### 流失原因分析

| 原因 | 权重 | 分析 |
|-----|-----|-----|
| 剧本数量少 | 高 | 当前仅 3 个示例剧本 |
| 信息不够吸引 | 中 | 描述过于简单，缺乏吸引力 |
| 选择困难 | 中 | 没有推荐/排序，用户不知如何选择 |
| 类型不符预期 | 中 | 剧本类型可能不符合用户偏好 |

#### 优化建议

1. **内容丰富化**
   - 增加剧本数量至 10+ 个
   - 按难度、时长、类型分类
   - 添加「编辑推荐」「热门榜单」

2. **信息展示优化**
   - 剧本预览（开头场景试读）
   - 用户评分/评价
   - 完成人数统计
   - 预计游戏时长更精准

3. **个性化推荐**
   - 基于用户偏好标签推荐
   - 「你可能喜欢」模块
   - 根据历史行为调整排序

4. **降低决策成本**
   - 「快速开始」按钮（随机推荐）
   - 标注「新手推荐」剧本

#### 数据埋点需求

```typescript
{
  event: 'script_list_view',
  scripts_viewed: string[], // 浏览过的剧本ID
  view_duration_ms: number,
  sort_method: 'default' | 'difficulty' | 'popularity',
  filter_used: boolean,
  exit_action: 'select_script' | 'back' | 'close'
}
```

### 节点 3：剧本详情流失

#### 场景描述
用户查看剧本详情后，未点击「开始游戏」就离开。

#### 流失率预估
**15-25%**

#### 流失原因分析

| 原因 | 权重 | 分析 |
|-----|-----|-----|
| 难度不明确 | 中 | 用户担心太难或太简单 |
| 时长担忧 | 高 | 用户不确定是否有足够时间 |
| 缺乏预览 | 高 | 无法预知游戏体验 |
| 角色信息不足 | 中 | 不知道能扮演什么角色 |

#### 优化建议

1. **时间管理辅助**
   - 显示「预计 12 分钟」并标注「可随时保存」
   - 提供「5 分钟体验版」选项

2. **预览功能**
   - 展示开场场景预览
   - 角色介绍卡片
   - 玩家评价/评分

3. **难度说明**
   - 可视化难度等级
   - 「适合新手」标签
   - 提示需要的选择数量

4. **社交证明**
   - 「已有 1,234 人完成」
   - 展示热门结局

#### 数据埋点需求

```typescript
{
  event: 'script_detail_view',
  script_id: string,
  time_on_page_ms: number,
  sections_viewed: ('description' | 'characters' | 'reviews')[],
  start_button_clicked: boolean,
  exit_reason: 'not_interested' | 'no_time' | 'too_hard' | 'other'
}
```

### 节点 4：开始游戏流失（关键节点）

#### 场景描述
用户点击「开始游戏」后，在游戏加载或第一个场景出现时离开。

#### 流失率预估
**10-20%**

#### 流失原因分析

| 原因 | 权重 | 分析 |
|-----|-----|-----|
| 加载等待 | 高 | 游戏初始化时间过长 |
| 首场景不吸引 | 高 | 开场内容缺乏吸引力 |
| 理解门槛 | 中 | 用户不知道如何操作 |
| 设备/网络问题 | 低 | 移动端兼容性问题 |

#### 优化建议

1. **加载优化**
   - 预加载剧本资源
   - 添加加载进度条 + 趣味文案
   - 首场景 SSR 渲染

2. **开场优化**
   - 设计高吸引力的开场场景
   - 前三句话抓住用户注意力
   - 避免大量文字堆砌

3. **新手引导**
   - 强化游戏指南（当前有 OnboardingGuide）
   - 第一次选择时高亮提示
   - 添加「跳过引导」选项

4. **容错处理**
   - 网络错误友好提示
   - 自动重试机制
   - 离线缓存支持

#### 数据埋点需求

```typescript
{
  event: 'game_start',
  script_id: string,
  load_time_ms: number,
  first_scene_shown: boolean,
  time_to_first_choice_ms: number,
  initial_scene_id: string,
  exit_before_first_choice: boolean
}
```

### 节点 5：游戏中途流失（核心流失点）

#### 场景描述
用户在游戏进行中离开，未完成剧本。

#### 流失率预估
**30-40%**（最关键的流失点）

#### 流失原因分析

| 原因 | 权重 | 分析 |
|-----|-----|-----|
| 内容疲劳 | 高 | 对话/场景重复感强，失去兴趣 |
| 选择焦虑 | 中 | 选项过多或意义不明确 |
| 时间不够 | 高 | 玩到一半有事离开 |
| 难度失衡 | 中 | 太难或太简单都可能导致流失 |
| 缺乏成就感 | 中 | 长时间没有正向反馈 |
| 移动端体验 | 中 | 小屏幕阅读体验不佳 |

#### 优化建议

1. **节奏控制**
   - 每 3-5 个场景设置小高潮
   - 穿插小成就/线索发现提示
   - 动态调整场景长度

2. **存档优化**
   - 自动保存（当前有 SaveButton）
   - 场景切换时自动存档
   - 「下次继续」快捷入口

3. **进度感知**
   - 显示进度条「当前进度 60%」
   - 分支树可视化（当前有 BranchTree）
   - 线索收集进度提示

4. **内容增强**
   - AI 生成更生动的对话
   - 添加场景背景图/音效
   - NPC 反应更个性化

5. **中断恢复**
   - 推送通知「继续你的冒险」
   - 存档列表展示游玩时长
   - 快速恢复上次进度

#### 数据埋点需求

```typescript
{
  event: 'game_session',
  script_id: string,
  session_id: string,
  scenes_completed: number,
  choices_made: number,
  time_played_ms: number,
  last_scene_id: string,
  exit_type: 'manual' | 'timeout' | 'error',
  save_created: boolean,
  // 场景级数据
  scene_events: [
    {
      scene_id: string,
      time_on_scene_ms: number,
      choice_made: boolean,
      hint_used: boolean
    }
  ]
}
```

### 节点 6：结局后流失

#### 场景描述
用户达成结局后，未进行分享、重玩或探索其他剧本。

#### 流失率预估
**40-50%**

#### 流失原因分析

| 原因 | 权重 | 分析 |
|-----|-----|-----|
| 完成感过强 | 高 | 用户觉得「任务完成」，没有继续动力 |
| 重玩动力不足 | 中 | 不知道有其他结局 |
| 分享动力不足 | 中 | 分享激励不够 |
| 无后续引导 | 高 | 结局页面缺乏下一步行动引导 |

#### 优化建议

1. **结局展示增强**
   - 结局卡片设计更精美
   - 显示达成率和稀有度
   - 解锁「结局图鉴」

2. **重玩激励**
   - 显示「还有 3 个结局等你发现」
   - 结局对比图
   - 「挑战完美结局」按钮

3. **社交激励**
   - 一键分享结局卡片
   - 邀请好友解锁专属剧本
   - 结局排行榜

4. **下一步引导**
   - 「探索更多剧本」推荐
   - 「挑战更高难度」
   - 「创作你的剧本」

#### 数据埋点需求

```typescript
{
  event: 'game_complete',
  script_id: string,
  ending_id: string,
  total_time_ms: number,
  scenes_visited: number,
  choices_total: number,
  achievements_unlocked: string[],
  share_clicked: boolean,
  share_channel: 'wechat' | 'feishu' | 'link' | null,
  replay_clicked: boolean,
  other_script_clicked: boolean,
  time_on_ending_page_ms: number
}
```

## 五、数据验证方案

### 5.1 数据采集架构

```
用户行为 → 前端埋点 SDK → 数据收集服务 → 数据仓库
                                          ↓
                                    分析看板/报警
```

### 5.2 核心仪表盘

#### 留存漏斗看板

```
首页访问 (100%)
  └─ 剧本列表 (70%)
      └─ 剧本详情 (55%)
          └─ 开始游戏 (45%)
              └─ 首次选择 (40%)
                  └─ 游戏中途 (25%)
                      └─ 达成结局 (18%)
                          └─ 再次游戏 (12%)
```

#### 场景级流失热力图

```
场景ID: start → scene_1 → scene_2 → scene_3 → ... → ending
流失率:  5%      8%        12%       15%              3%
```

### 5.3 A/B 测试方案

| 测试项 | 对照组 | 实验组 | 成功指标 |
|-------|-------|-------|---------|
| 首页引导 | 无引导 | 强制引导 | 首页→开始游戏转化率 +10% |
| 存档提示 | 手动存档 | 自动存档+提醒 | 游戏中途留存 +5% |
| 结局页面 | 仅显示结局 | 结局+推荐剧本 | 结局后重玩率 +15% |

### 5.4 数据验证步骤

1. **基线数据采集**（1-2周）
   - 部署所有埋点
   - 收集现有用户行为数据
   - 建立流失漏斗基线

2. **假设验证**（2-3周）
   - 分析各节点实际流失率
   - 验证流失原因假设
   - 调整优化优先级

3. **优化迭代**（持续）
   - 按 P0 → P1 → P2 顺序优化
   - 每项优化后观察数据变化
   - 快速迭代验证

## 八、监控与报警

### 8.1 关键指标监控

```yaml
# 指标阈值配置
metrics:
  next_day_retention:
    warning: 35%
    critical: 25%
  
  game_completion_rate:
    warning: 50%
    critical: 40%
  
  average_session_time:
    warning: 8min
    critical: 5min
```

### 8.2 报警机制

- 次日留存连续 3 天低于预警值 → 发送告警
- 游戏完成率突降 10%+ → 紧急排查
- 特定场景流失率异常 → 自动标记分析
